# RED App Uninstaller

Графический деинсталлятор для РЕД ОС в стиле окна «Программы и компоненты» Windows.

## Что уже работает

- список приложений по **настоящим названиям из системных ярлыков `.desktop`**;
- отображение штатных значков приложений;
- поиск по названию, описанию, пакету и альтернативным ярлыкам;
- сопоставление ярлыка с реальным RPM-пакетом через `rpm -qf`;
- поддержка Flatpak-приложений;
- объединение нескольких ярлыков, принадлежащих одному пакету;
- просмотр пакета, версии, размера, даты установки и путей ярлыков;
- предварительный расчёт операции `dnf remove --assumeno`;
- удаление RPM через `pkexec dnf -y remove` с системным окном авторизации;
- удаление Flatpak штатной командой `flatpak uninstall`;
- защита нескольких критических системных пакетов от удаления через интерфейс.

## Установка в РЕД ОС

Распакуйте архив, откройте терминал в каталоге проекта и выполните:

```bash
chmod +x install.sh
./install.sh
```

Установщик сам запросит права администратора. После установки приложение появится в системном меню под названием **«Удаление программ»**.

Для запуска без установки:

```bash
sudo dnf install python3-gobject gtk3
python3 red_uninstaller.py
```

## Удаление самого приложения

```bash
chmod +x uninstall.sh
./uninstall.sh
```

## Как определяется программа

1. Сканируются видимые ярлыки в `/usr/share/applications`, `/usr/local/share/applications` и пользовательских каталогах.
2. Из ярлыка читаются локализованные поля `Name`, `GenericName`, `Comment`, `Icon` и `Exec`.
3. Владелец ярлыка определяется командой `rpm -qf`.
4. Если пользовательский ярлык не принадлежит RPM, приложение пытается определить пакет по исполняемому файлу из `Exec`.
5. Для экспортированных Flatpak-ярлыков используется идентификатор `X-Flatpak`.

Благодаря этому пользователь видит, например, **«Графический редактор GIMP»**, а не только техническое имя RPM-пакета.

## Ограничения MVP

- AppImage и программы, вручную распакованные в `/opt`, не имеют универсального механизма удаления и пока не показываются.
- Wine-программы пока не индексируются.
- Размер RPM — размер установленного пакета из RPM-базы, а не точный объём всех пользовательских данных.
- Предварительный расчёт DNF показывается как штатный текст транзакции.
- Интерфейс не удаляет пользовательские настройки из домашнего каталога.

## Проверка исходника

```bash
python3 -m py_compile red_uninstaller.py
python3 red_uninstaller.py --self-test
```

## Сборка RPM

Для сборки потребуется `rpm-build`:

```bash
sudo dnf install rpm-build
chmod +x build-rpm.sh
./build-rpm.sh
```

Готовый пакет появится в `~/rpmbuild/RPMS/noarch/`.

## Исправления версии 0.1.3

- явно выбирается Gdk 3.0 до импорта GTK-компонентов;
- вызывается `window.show_all()`, поэтому элементы интерфейса GTK3 отображаются, а не остаётся пустое белое окно.

## Исправления и возможности версии 0.1.3

- приложение ищет `.desktop`-ярлыки в каталоге рабочего стола из `user-dirs.dirs`, а также в `~/Desktop` и `~/Рабочий стол`;
- найденные ярлыки отображаются в окне сведений;
- перед удалением доступен включённый по умолчанию пункт удаления ярлыков;
- ярлыки удаляются только после успешного удаления пакета;
- удаляются только сопоставленные `.desktop`-файлы непосредственно из каталогов рабочего стола текущего пользователя.
